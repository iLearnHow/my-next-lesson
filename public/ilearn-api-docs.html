iLearn.how API Documentation

The most sophisticated educational content API ever built. Generate age-appropriate lessons in any language with professional narration and video.

What is iLearn.how?

iLearn.how provides a powerful REST API that delivers personalized educational content. With a single API call, you can generate:

  • Age-intelligent lessons - Content automatically adapts for ages 2 to 102
  • Cultural adaptation - True localization beyond translation
  • AI personalities - Choose from Grandmother, Fun, or Neutral tones
  • Rich media - Professional audio narration and video included
Private Beta: iLearn.how is currently in private beta. Request access to get your API key and start building.

Key Features

Age Intelligence

Vocabulary, complexity, and examples automatically adapt for any age from toddlers to seniors.

Cultural Adaptation

15+ languages with true cultural intelligence - not just translation but cultural context.

AI Personalities

Three distinct teaching personalities that authentically adapt to the learner's age and needs.

Production Ready

Every lesson includes professional AI-generated audio and video - no additional production needed.

Base URL

https://api.ilearn.how/v1

Quick Start

Get up and running with the iLearn.how API in minutes.

1. Get Your API Key

Sign up for our private beta to receive your API key. API keys are provided in the following format:

ilearn_live_sk_1234567890abcdef...
Private Beta: Currently accepting applications for developer access. Apply for API access to get started.

2. Install SDK (Optional)

For the fastest setup, install our JavaScript SDK:

npm install ilearn-sdk

Or use any HTTP client with our REST API directly.

3. Make Your First Request

Get a personalized daily lesson with a simple API call. Here are examples in different languages:

Using cURL

curl -H "Authorization: Bearer YOUR_API_KEY" \ "https://api.ilearn.how/v1/daily-lesson?age=25&tone=fun&language=english"

Using JavaScript SDK

import { iLearn } from 'ilearn-sdk'; const ilearn = new iLearn('YOUR_API_KEY'); const lesson = await ilearn.getDailyLesson({ age: 25, tone: 'fun', language: 'english' }); console.log(lesson);

Using Python

import requests headers = {'Authorization': 'Bearer YOUR_API_KEY'} params = { 'age': 25, 'tone': 'fun', 'language': 'english' } response = requests.get( 'https://api.ilearn.how/v1/daily-lesson', headers=headers, params=params ) lesson = response.json() print(lesson)

Authentication

All API requests require authentication using your API key.

API Key Authentication

iLearn.how uses API key authentication. Include your API key in the Authorization header using Bearer authentication:

Authorization: Bearer ilearn_live_sk_1234567890abcdef...

API Key Types

We provide different types of API keys for different use cases:

И Test Keys

ilearn_test_sk_...

For development and testing. No charges applied, includes synthetic lesson data.

Live Keys

ilearn_live_sk_...

For production use. Generates real lessons with professional audio/video.

Making Authenticated Requests

Here's how to include authentication in different programming languages:

cURL

curl -H "Authorization: Bearer YOUR_API_KEY" \ -H "Content-Type: application/json" \ "https://api.ilearn.how/v1/daily-lesson"

JavaScript (Fetch)

const response = await fetch('https://api.ilearn.how/v1/daily-lesson', { headers: { 'Authorization': 'Bearer YOUR_API_KEY', 'Content-Type': 'application/json' } }); const lesson = await response.json();

Python (Requests)

import requests headers = { 'Authorization': 'Bearer YOUR_API_KEY', 'Content-Type': 'application/json' } response = requests.get( 'https://api.ilearn.how/v1/daily-lesson', headers=headers )

Node.js (Axios)

const axios = require('axios'); const config = { headers: { 'Authorization': 'Bearer YOUR_API_KEY', 'Content-Type': 'application/json' } }; const response = await axios.get( 'https://api.ilearn.how/v1/daily-lesson', config );

API Key Management

Security Best Practices:
  • Never expose API keys in client-side code
  • Use environment variables to store keys
  • Rotate keys regularly
  • Use test keys for development

Environment Variables

Store your API keys securely using environment variables:

# .env file ILEARN_API_KEY=ilearn_live_sk_your_actual_key_here # In your code const apiKey = process.env.ILEARN_API_KEY;

Key Rotation

You can generate new API keys and revoke old ones through your developer dashboard. Best practices:

  • Rotate keys every 90 days
  • Immediately revoke compromised keys
  • Test new keys before revoking old ones
  • Update all applications when rotating keys

Authentication Errors

Authentication errors return specific HTTP status codes and error messages:

401 Unauthorized

{ "error": { "code": "unauthorized", "message": "Invalid API key provided", "type": "authentication_error", "documentation_url": "https://docs.ilearn.how/authentication" } }

403 Forbidden

{ "error": { "code": "insufficient_permissions", "message": "API key does not have permission for this resource", "type": "permission_error", "required_permission": "lessons:read" } }

Rate Limiting by Key Type

Different API key types have different rate limits:

Test Keys

  • 100 requests/hour
  • 10 requests/minute
  • Burst limit: 20 requests

Live Keys

  • Unlimited requests
  • Burst limit: 100 requests
  • Fair usage policy applies

Daily Lesson API

Get a personalized daily lesson for any age, tone, and language.

Endpoint

GET /v1/daily-lesson
https://api.ilearn.how/v1/daily-lesson

Returns a complete educational lesson with scripts, audio, video, and metadata tailored to the specified learner characteristics.

Query Parameters

Parameter Type Required Description
age integer Required Learner's age (2-102). Content automatically adapts vocabulary, complexity, and examples.
tone string Required Teaching personality: grandmother, fun, or neutral
language string Optional Target language with cultural adaptation (default: english)
cultural_context string Optional Cultural adaptation context (auto-detected if not provided)
include_media boolean Optional Include audio/video URLs (default: true)
lesson_date string Optional Specific date for lesson (YYYY-MM-DD). Defaults to today's lesson.

SDK Examples

JavaScript SDK

import { iLearn } from 'ilearn-sdk'; const ilearn = new iLearn(process.env.ILEARN_API_KEY); // Basic lesson request const lesson = await ilearn.getDailyLesson({ age: 25, tone: 'fun', language: 'english' }); // Access lesson components console.log(lesson.lesson_metadata.title); console.log(lesson.audio_url); console.log(lesson.scripts.length); // Handle errors try { const lesson = await ilearn.getDailyLesson({ age: 8, tone: 'grandmother', language: 'spanish' }); } catch (error) { if (error.code === 'unsupported_language') { console.log('Try a different language'); } }

Python SDK

from ilearn import iLearn ilearn = iLearn(api_key=os.environ['ILEARN_API_KEY']) # Basic lesson request lesson = ilearn.get_daily_lesson( age=25, tone='fun', language='english' ) # Access lesson components print(lesson['lesson_metadata']['title']) print(lesson['audio_url']) print(len(lesson['scripts'])) # Handle errors try: lesson = ilearn.get_daily_lesson( age=8, tone='grandmother', language='spanish' ) except iLearn.UnsupportedLanguageError as e: print(f"Language not supported: {e.language}") except iLearn.InvalidParameterError as e: print(f"Invalid parameter {e.parameter}: {e.message}")
Pro Tip: Cache lessons by age_tone_language_date combination to reduce API calls for the same content. Daily lessons change once per day.
